Adding a Character

This guide will focus on teaching you how to add a character from the Standard Assets. If you choose to find your own on the Asset Store be wary. Some of those character models might be buggy, their code might not work right, or the animator might be sloppily put together.

While third-party companies put out a wide range of awesome assets for you to download, not all of them are expertly assembled. While errors happen here and there for our students, beware using outside character models until you become familar with your code

Let's add a basic character for now and we can customize it.

ThirdPersonController

Go to Standard Assets → Characters → ThirdPersonCharacter → Prefabs


Place the ThirdPersonController.prefab in your scene where you want to start when you begin the game.

Worldbuilding Tip:

Place your character with purpose. Instead of just dropping the character in the world, create a purpose, or a quest. This is when you can use your terrain to guide the character to a boss, loot, or other npcs (non-player characters) to interact with.

Temple Run made with Unity


Rename the ThirdPersonController to PlayerController

Before we test it though, we want to make sure we can see our character, so let's add our camera

Camera

Our scene came with a camera, but we’re going to replace it with a better one that we can control with the mouse. First, click on the Main Camera that came with our scene and press the Delete key on the keyboard to delete it. To add the new one:

Standard Assets → Cameras → Prefabs

Place the FreeLookCameraRig in your scene


In order to play the game, Unity needs to know which camera to use. We need to tell it this is the Main Camera. We’ll do this with a tag. Select the FreeLookCameraRig and check the Inspector. Where it says Untagged, click and change that to MainCamera.


Look at the FreeLookCameraRig’s script component. It is set to Auto-target Player. This is good, but we need to tell it who the player is. Select the PlayerController and tag it as Player.

Now the camera will follow the player! We can adjust other settings on the Camera. The most important are:

Move Speed - how fast the camera moves to follow the player.

Turn Speed - how fast the camera moves with the mouse.

Lock Cursor - keep checked

When testing, the cursor will be locked. To unlock it so you can stop or pause the game, press Escape

Testing the Game

Try playing the game by pressing the play button above the scene view! You can move around with the arrow keys or WASD and jump with the spacebar. Now you can explore the realm you created.

You can also adjust the movement of the character. Under the Third Person Character script component on the PlayerController, in the Inspector, you can adjust various settings:

Moving Turn Speed - how fast the character turns while in motion. Default 360.

Stationary Turn Speed - how fast the character turns while standing still. Default 180.

Jump Power - how high/far the character jumps. Default 6.

Gravity Multiplier - the force of gravity on the player. Default 1.

Run Cycle Leg Offset - Adjusts the running animation. Leave alone for now.

Move Speed Multiplier - how fast the character runs. Default 1.

Anim Speed Multiplier - how fast the character’s animations play. Should be adjusted when speed is increased to look more natural.

Ground Check Distance - how close the character needs to be to the ground to be “grounded”.

Adjust these settings until the character moves how you like!


Frequently test your game to make sure that there are no bugs or errors. It's easier to pin-point problems if you know it was the last script you worked on, rather than testing after adding your 10th character and trying to find out where the errors are coming from.

Swapping out your character

Up until now, we have been using the same character called Ethan. Now is a good time to go into the Asset Store and find a character model that we can use.

Remember how earlier we mentioned that not all character models are built the same and some have bugs? This is how we can work around that.

Find a character you like and download it, we will be working on it in the next section

Look for your newly downloaded/imported characters. They will be a new folder in the root of the project - the Assets folder. Your files will be in there. Go find the prefab for your object. It should be a picture of the object with a grey background, probably in a folder called prefabs.

First, click the arrow next to the PlayerController in the Hierarchy to open up its child objects.

The PlayerController is an object that controls all of its child objects. It applies the control settings on it to the model attached to it.

We don’t want any of Ethan’s parts, so go ahead and highlight all three of the Ethan objects and delete them. This is basically the shell of the character. You will need to open the Prefab instance and delete them from there. Then click the back arrow next to PlayerController to exit the Prefab mode.


Now the controller needs a new model to control. Drag the model onto the PlayerController in the hierarchy to make it a child.


Next, we want to make sure that it is setup to use humanoid animations. We are taking the default animations from Ethan, the default character, and using them with this new model. As Ethan is a human, we need to make sure that this model can use humanoid animations. Double-click on your model in the Project View (probably in a folder called models) to open it in the Inspector.

These are the import settings for the model. Click on Rig.

Set Animation Type to Humanoid, and make the avatar from the model.

Now we want to swap out our avatars. Click on the arrow next to your model:

You should see an Avatar. When a rigged model is imported into Unity, it creates an avatar that customizes the animations for that model.

Currently, our controller is set to use Ethan’s avatar. Click on the PlayerController in the Hierarchy and see that in the Inspector.

Drag your new Avatar where it says EthanAvatar. Now play your game with your new character!
